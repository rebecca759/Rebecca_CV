Shoud We Use Relative Size When Styling an HTML Element ?

Using absolute length units like pixels can be easy. However, if a user changes the default font-size of browser or device, the header’s font-size (say 20px) will remain 20 px. Hence user’s font preferences won’t be reflected, which is not a good user experience. Thankfully, setting font-sizes in pixels doesn’t completely ruin accessibility. The user can still zoom in and out with ctrl plus +/-. However, web developers can do better as zooming in or out is not very convenient for the end user.

Using relative size like EM and REM gives us flexibility in our designs, and the ability to scale elements up and down, instead of being stuck with fixed sizes. We can use this flexibility to make our designs easier to adjust during development, more responsive, and to allow browser users to control the overall scale of sites for maximum readability.

Responsive design can help solve a lot of problems for websites. It will make sites mobile-friendly, improve the way it looks on devices with both large and small screens, and increase the amount of time that visitors spend on the site. It can also help improve rankings of the website in search engines.

All in all, although using absolute size units is easier, it’s very important to consider using relative size to create a good experience for every user and increase the number of users visiting our site.

How is a Font-Size other than the Default Interpreted ?

The default browser font-size is 16px, it’s the also the same thing as setting the font-size to 100% because the % value relates to the default browser font size. All HTML tags inherit a font-size of 16px. If we set the font-size to 200px, it will be interpreted as being 12.5 times the body’s size or 11.5 times bigger than the body.

New Features of CSS3

Pseudo-classes: A few more pseudo-class selectors were added in CSS3. One is the :root selector, which allows designers to point to the root element of a document. In addition, as a complement to the :first-child selector, the :last-child was added. With it one can select the last element named of a parent element. A functional notation for selecting specified elements that fail a test has also been created. The negation pseudo-class selector, :not can be coupled with almost any other selector that has been implemented.

CSS3 Colors: The color keyword list has been extended in the CSS3 color module to include 147 additional keyword colors. CSS3 also provides us with a number of other options: HSL, HSLA, and RGBA. RGBA works just like RGB, except that it adds a fourth value: alpha, the opacity level or alpha transparency level. HSL stands for hue, saturation, and lightness. With HSL you can tweak either just the saturation or the lightness while keeping the same base shade.

Rounded Corners: The border-radius property lets you create rounded corners without the need for images or additional markup.

Box Sizing: It doesn't feel at all intuitive to have the CSS width and height of an element affected by its padding and borders. It can set it to border box, which makes elements behave exactly the way you expect.

Linear Gradients: W3C added the syntax for generating linear gradients with CSS3.

Multiple Background Images: In CSS3, there’s no need to include an element for every background image; it provides us with the ability to add more than one background image to any element.

Drop Shadows: CSS3 provides the ability to add drop shadows to elements using the box-shadow property. This property lets you specify the color, height, width, blur, and offset of one or multiple inner and/or outer drop shadows on your elements.
